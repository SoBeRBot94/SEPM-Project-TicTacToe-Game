class Player.Player(typePlayer, difficulty='default')¶

Bases: object

The class which controls if it’s a human player or an AI. Can take two parameters as input, where the type of the player (AI or user) is mandatory A second parameter, difficulty, is not needed when initializing a user type of player but it is mandatory when initializing a AI. If trying to initialize an AI without the difficulty parameter, a ValueError is raised from the _AI class. If trying to initialize the Player class without specifying that the player is a ‘ai’ or a ‘user’, a ValueError is raised notifying the client of this. The only public facing method which is used by the client is nextMove.

class AI._AI(difficulty)¶

Bases: object

Class which is used to initialize an AI type of player. Shouldn’t be called directly, instead, call it via the class Player. The class _AI, which receives the difficulty parameter from the initialization of the Player class, uses it to be able to initialize the correct difficulty of the AI. If the difficulty parameter isn’t one of the following: hard, medium or easy, a ValueError is raised notifying the client of this.
